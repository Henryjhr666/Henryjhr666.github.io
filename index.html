第二篇

2024年10月29日

接着今天凌晨的内容继续写点吧，这几天写的都是之前初始C语言学的，想要把笔记慢慢码到博客上……

正文【初识C语言】

char // 字符数据类型
short // 短整型
int // 整型
long // 整型
longlong // 更长的整型
double // 双精度浮点型
float // 单精度浮点型
例：double price=66.6; 精度高于 float price=66.6;

计算机中的单位 // bit、byte、kb、mb、gb、tb、pb……
c语言规定： size of (long) >=size of (int)

变量→局部和全局// 大概就理解成“{ }”外面是全局，里面是局部变量；【全局变量和局部变量名字相同的情况下，局部优先，建议不要将全局变量和局部变量名字一样】
#define _CRT_SECURE_NO_WARNINGS //用的vs2022里面要是不写这一行，下面使用scanf输入函数会报错，输出报错栏里面显示有
#include<stdio.h>
int main()
{
int num1=0;
int num2=0;
scanf{“%d %d”,&num1,&num2}; // scanf输入函数 &取地址符【vs2022里面输入函数“scanf_s”是vs编译器自己提供的输入函数，非标准c提供的函数，其他的编译器认识】
int sum = num1+num2;
printf(“%d\n”,sum);
return 0;
}
// 在vs安装路径下有一个文件：newc++file.cpp文件，在vs工程中创建新的c.或cpp文件时，都是拷贝newc++file.cpp这个文件的

变量的作用域→局部变量/全局变量
局部变量：变量所在局部范围
全局变量：作用域是整个工程
例：
int main()
{
{
int a=10;
printf(“%d\n”,a);
}
printf(“%d\n”,a); //这一行会报错“未声明的标识符”
return 0；
}
// 声明来自外部的符号“exturn int a=0；”（来自编译器中别的文件）

变量的生命周期→局部变量/全局变量
局部变量：进入作用域生命周期开始，出作用域生命周期结束。
全局变量：整个程序的生命周期。

常量{“字面常量”“const修饰的常变量”“#define定义的标识符常量”“枚举常量”}
// 字面常量 例：30；3.14；‘w’；//字符”abc”
//const修饰的常变量 const int a=10；//a是const修饰，本质是变量，但不能被修改，有常量的属性
//define修饰的标识符常量
例：
#define MAX 100 //MAX就是被define定义的标识符常量
int main()
{
printf(“%d\n”,MAX);
int a=MAX;
printf(“%d\n”,a);
return 0;
}
//枚举常量 public enum Day {
MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
} //enum（枚举）

字符串+转义字符+注释
char //字符类型 char ch=’w’; 注：字符串的结束标志是一个”\0”的转义字符（转变原来字符的意思），计算字符串长度时”\0”是结束标志，不算作字符串内容。
char arr1[10]=”abcdef”; //七个字符 char[7]
char arr2[]={‘a’,’b’,’c’,’d’,’e’,’f’};//char[6]
printf(“%s\n”,arr1); //输出abcdef
printf(“%s\n”,arr2); //输出abcdef烫烫烫烫烫烫abcdef
字符串 //”“双引号引起来的【C语言中没有字符串类型】
// “\“转义为一个”"打印，”\0”即输出”\0”
//strlen(); //求字符串长度的库函数，string length→#include<string.h>
int main()
{
int len = strlen(“abc”);
printf(“%d\n”,len); //输出：3
return 0;
}
%d 打印整型
%c 打印字符
%s 打印字符串
%f 打印float类型的数据
%lf 打印double类型的数据
%zu 打印size of返回值

转义字符
\a 响铃 007
\b 退格 008
\f 换页 012
\n 换行 010
\r 回车 013
\t 水平制表 009
\v 垂直制表 011
\ 反斜杠 092
\? 问号字符 063
\‘ 单引号字符 039
\‘’双引号字符 034
\0 空字符；结束的标志 000
\ddd 任意字符 三位八进制
\xdd 任意字符 二位十六进制

例：
printf(“%c\n”,’\130’); //输出：X 八进制130→换算成十进制=88 ASCII编码中88→X
printf(“%c\n”,’\x60’); //输出：、 十六进制的60→换算成十进制=96 ASCII编码中96→、

注释：//……或者”/* /“
/ int a= 10;/ →C语言注释风格，不支持嵌套注释，遇到/就结束
// int a=10；// →c++的注释风格

int *p=NULL；//创建指针变量p，并赋值为NULL

没有学markdown语法，编辑这个的格式被搞的好烦，还是要去学一下，嗯要是能够vx两个窗口的编辑的同时就能看见效果就最好了……

在bilibili学 c语言的时候，看见up主提到了写博客，去了解了一下，想着自己也来试着写写，但又不知道写些啥，昨天搭建好了hexo博客，那么今天就先从写自己的笔记开始吧！“真正意义上的第一篇”

正文【初始C语言】

计算机语言的发展是一个由低级向高级演变的过程，二进制指令→汇编指令（助记符）→B语言→C语言（高级语言）
C语言国际标准ANSI C
编译器主要有Clong/GCC/WIN-TC/MSVC/Turboc c++是一门编译型语言
test.c文件→编译→链接→test.exe文件 .c源文件（注意后缀）如果是.c++，编译器会按照c++语法来编译代码，在后来的实操中确实有一次忘记修改，然后报错了，但是个人感觉还是蛮罕见的
#include <stdio.h> //标准输入输出头文件，可以看成是”向库打招呼“
int main() //C语言中一定要有main函数 main函数是程序的入口，有且仅有一个 //有些教材中也会有int main(void)→这里的void就是明确没有参数
{
printf(“hello world\n”); //printf是库函数，专门用来打印数据
return 0; //试了别的数字，一样可以运行，返回的是0，对应一开头的int ，试过用字母，定义了int z=0;return z; 如果没有这一行，会运行不了，但是vs中没有可以运行因为编译器编译的时候加上了这一行……
}
//上面应该就是每个人初始C语言敲出来的第一份代码吧~
vs2022运行的时候是 ctrl+f5，不如dev c++的f11来得方便，但是dev c++里面中文会识别成乱码，问了gpt还是没有搞好，蠢了点……

结语
