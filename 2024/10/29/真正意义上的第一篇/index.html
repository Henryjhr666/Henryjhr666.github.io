真正意义上的第一篇

2024年10月29日

在bilibili学 c语言的时候，看见up主提到了写博客，去了解了一下，想着自己也来试着写写，但又不知道写些啥，昨天搭建好了hexo博客，那么今天就先从写自己的笔记开始吧！“真正意义上的第一篇”
可能看上去很无聊，但是我确实是要把之前的笔记上传到这里，也就是一些刚开始学习c语言时候写的笔记，嗯是这样……

正文【初始C语言】

计算机语言的发展是一个由低级向高级演变的过程，二进制指令→汇编指令（助记符）→B语言→C语言（高级语言）
C语言国际标准ANSI C
编译器主要有Clong/GCC/WIN-TC/MSVC/Turboc c++是一门编译型语言
test.c文件→编译→链接→test.exe文件 .c源文件（注意后缀）如果是.c++，编译器会按照c++语法来编译代码，在后来的实操中确实有一次忘记修改，然后报错了，但是个人感觉还是蛮罕见的
#include <stdio.h> //标准输入输出头文件，可以看成是”向库打招呼“
int main() //C语言中一定要有main函数 main函数是程序的入口，有且仅有一个 //有些教材中也会有int main(void)→这里的void就是明确没有参数
{
printf(“hello world\n”); //printf是库函数，专门用来打印数据
return 0; //试了别的数字，一样可以运行，返回的是0，对应一开头的int ，试过用字母，定义了int z=0;return z; 如果没有这一行，会运行不了，但是vs中没有可以运行因为编译器编译的时候加上了这一行……
}
//上面应该就是每个人初始C语言敲出来的第一份代码吧~
vs2022运行的时候是 ctrl+f5，不如dev c++的f11来得方便，但是dev c++里面中文会识别成乱码，问了gpt还是没有搞好，蠢了点……

结语

就先写这么多吧，困觉了……2024.10.29 00：47
